Example Query API Form

Interrogating: http://localhost:8080/Query/search

For detailed info: https://github.com/HuntsmanCancerInstitute/Query

Query form params:

Retrieve the currently supported options and filters.
     FetchOptions: true or false

Instruct the server to not only perform the index query (fast) but also pull any of the associated records from disk.
     FetchData: true or false or force

If a vcf file is used in the query, any vcf records found to intersect will be filtered for those matching chr, pos, ref, and at least one alt. Other intersecting records (e.g. bed or maf) will be returned without additional checks, only vcf.
     MatchVcf: true or false

Return the file headers for records found to intersect your query.
     IncludeHeaders: true or false

Provide one or more java regular expressions, no spaces, delimited with a semi-colon. Returning data paths must match ONE of these.
Use to limit returning data to a particular set of projects, file types, analysis types, etc, e.g. \.vcf\.gz;\.maf\.txt\.gz
Note, a .* is appended to the front and back of each regEx.
     Reg Ex One:

Provide one or more java regular expressions, no spaces, delimited with a semi-colon. Returning data paths must match ALL of these.
Use to require returning data match a particular genome build, be from a specific file type, or from a particular file, e.g. /B37/
Note, a .* is appended to the front and back of each regEx.
     Reg Ex ALL:

Choose a normalized vcf or bed file to upload and intersect with the Query search engine, gz/zip OK. For vcf, be sure to vt normalize and decompose_blocksub, see https://github.com/atks/vt.
